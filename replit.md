# Atlas — Auth Hardening and Automatic Orphan Recovery

## Overview
Atlas is a desktop-first personal productivity command center designed as a local-first capture intelligence system. It efficiently processes user input by prioritizing cost-effective local parsing methods and using AI services like Claude Haiku only as a fallback for ambiguous cases, significantly reducing AI costs. The project includes a comprehensive captures tRPC router for parsing, previewing, logging, and statistical analysis, a re-engineered capture modal, email-to-inbox functionality, and a robust, authenticated application shell focused on user experience. It aims to be a command center for personal productivity.

## System Architecture
The application is built on Next.js 15 (App Router) + React 19 with TypeScript. Styling uses Tailwind CSS 3.4 and custom Stratum tokens, complemented by UI primitives from Radix UI, cmdk, vaul, and sonner. Dark mode is supported via `next-themes`.

**Core Architectural Decisions:**
- **Hybrid Parsing Pipeline**: Utilizes `chrono-node`, regex, and `compromise.js` for local-first input parsing, with Claude Haiku as a fallback for ambiguous cases to optimize AI costs.
- **tRPC API**: Provides a type-safe API layer for all backend interactions.
- **Database**: PostgreSQL via Prisma, featuring a foundational schema for users, audit logs, integration tokens, AI call logs, Notes, NotesFolder, and a generic Link graph table.
- **Authentication**: Clerk (`@clerk/nextjs` v7) handles all authentication, mapping Clerk user IDs to Prisma `User` rows via `clerk_id`.
- **Marketing Pages**: Public-facing marketing homepage at `/`, `/privacy`, and `/terms`. Authenticated users visiting `/` are redirected to `/tasks`.
- **Application Shell**: An authenticated shell featuring a `TwoPaneLayout`, `ModuleSwitcher`, `TopBar`, `CommandPalette` (`⌘K`), and `KeyboardShortcutsOverlay` (`⌘/`).
- **Capture Modals**: A re-engineered `CaptureModal` (`⌘⇧I`) for efficient input capture.
- **Settings and Configuration**: Comprehensive settings for managing profile, appearance, preferences (locale/formatting), capture preferences, integrations, AI, backups, data, account, and system (Jobs Management). The Settings → System page lists all registered background jobs, with cards showing name, description, human-readable schedule, status (Active/Paused), last-run outcome and result, next-run estimate, and action buttons (Run now / Pause / Resume). The `jobs` tRPC router (`src/server/routers/jobs.ts`) backs this UI with `jobs.list`, `jobs.runNow`, `jobs.pause`, and `jobs.resume` procedures. Audit log entries are written for `job_paused` and `job_resumed` actions.
- **Locale Formatting Layer**: `src/core/locale/` provides pure `formatDate`, `formatNumber`, `formatCurrency`, `formatTime`, `formatDateTime`, `formatRelativeDate`, `formatWeekdayFull`, `formatWeekdayAbbrev`, `formatMonthAbbrev` functions and a `useLocale()` React hook. Supports Pakistan (PKR), US (USD), UK (GBP), and Custom presets. All date/number/time/currency displays throughout the app use locale-aware formatters. `LocaleSettings` now includes a `language` field (e.g. `"en"`, `"ar"`, `"fr"`) that maps to a date-fns locale object, enabling localized weekday and month names (Monday → الاثنين in Arabic, etc.). The Settings → Preferences page exposes a Language selector alongside the locale preset dropdown with 20 language options, and the live preview shows a Weekday row that updates in real-time.
- **Email-to-Inbox**: Integration of email parsing (`mailparser`) with Resend inbound webhooks for capturing information from emails, including attachment handling and configurable filtering.
- **Date Handling**: Timezone-aware date utilities using `date-fns-tz`.
- **Google Drive Integration**: Utilizes the Google Drive API for linking and encrypting Drive tokens, with a 4-step OAuth wizard.
- **Queueing**: A priority-aware in-memory dispatch queue with DB-backed rate limiting. `pg-boss` (v10) provides a durable scheduled-job runner for cron jobs like `drive-sync-notes`, `drive-sync-tables`, `session-cleanup`, `trash-retention`, and `attachment-cleanup`.
- **Error Handling & Logging**: Comprehensive audit logging and Pino for structured logging.
- **Extensible Command and Shortcut Registries**: Context-based registries for managing application commands and keyboard shortcuts.
- **UI/UX Decisions**: Desktop-first design, component splitting, `React.memo` for performance, and narrowed Prisma `select` statements for efficiency.
- **Tasks Features**: Includes `ChecklistItem` model, `Checklist tRPC router`, `ChecklistSection` component, `SubtaskSection` and `SubtaskRow` components, updated task inspector with breadcrumbs, checklist progress badges in task list items, and enforced task depth.
- **Notes Editor Infrastructure (Wave 4a Phase 3)**: TipTap (ProseMirror) editor core built in `src/core/editor/` and `src/components/notes/`. Includes: `tiptap-config.ts` (extensions: StarterKit, Link, CodeBlockLowlight/lowlight, TaskList, Underline, Strike, Placeholder), `reference-extension.ts` (custom ProseMirror plugin for `[[note]]`, `#tag`, `@context` triggers), `slash-command-extension.ts` (`/` block-type command menu), `markdown-export.ts` (TipTap JSON→Markdown), `markdown-import.ts` (Markdown→TipTap JSON), `text-extraction.ts` (JSON→plain text for FTS). Components: `reference-picker.tsx` (searchable tRPC-backed dropdown with create-note option), `slash-command-menu.tsx` (10 block commands), `note-editor.tsx` (full editor wrapper with 1s debounced auto-save, Cmd+S, save status indicator, image-paste upload, URL-paste-as-link). Notes tRPC router at `src/server/routers/notes.ts` (list, get, create, update, delete, search).
- **Project Enhancement (Wave 4a Phase 5)**: Projects now have Type (project/goal/habit) and Target Date fields, surfaced in the UI. The `projects.create` and `projects.update` tRPC procedures accept `type` (enum: project|goal|habit) and `target_date` (ISO datetime). Audit log actions added: `project_type_changed` and `project_status_changed`. Components in `src/components/projects/`: `project-type-selector.tsx` (dropdown with emoji icons), `project-status-selector.tsx` (inline status changer), `project-target-date-picker.tsx` (popover date picker), `project-brief-display.tsx` (shows `is_project_brief` note with excerpt), `project-notes-section.tsx` (lists all project notes with "+ New note" button). The `ProjectDetailHeader` was updated to show the inline type/status/target-date selectors and dims when status is completed/dropped. The Tasks sidebar groups root projects into collapsible "Projects/Goals/Habits" sub-groups with independent localStorage collapse states; the "+" button is now a dropdown with "New Project/Goal/Habit" options. The `ProjectAddForm` accepts a `defaultType` prop and shows the type picker. The project detail page shows the brief above the task list and the notes section below it.
- **Project Type System Rework (Wave 4a)**: `Project.type` is now a free-form string (1-32 chars, alphanumeric + spaces + hyphens, stored lowercase) with no DB-level enum or check constraint. "Habit" is no longer a first-class type (existing Habit projects migrated to Goal). Key additions: `src/core/projects/type-validation.ts` (validate/normalize/capitalize), `src/core/projects/time-distance.ts` (natural-language time deltas), `src/core/projects/type-suggestions.ts` (adaptive curated suggestions). The sidebar now groups projects dynamically by type (most-used first, then alpha), with per-type collapse state in localStorage. The `[+]` button opens a custom picker menu with Core types, Your types, and Custom type option. Custom type dialog validates input and deduplicates case-insensitively. The All Projects page (`/tasks/projects`) has filter pills with `?type=` URL state and "More ▼" overflow for 4+ types. Project detail header shows a metrics row: N tasks total · M active · K completed, target date with natural time-distance, and last activity (omitted if older than 30 days). New tRPC procedure `projects.distinctTypes()` returns `{type, count}[]` for all types. `projects.list` accepts optional `type` filter parameter.
- **Tables Module (Wave 4b)**: Full structured-data module peer to Notes. Prisma models: `Table`, `TablesFolder`, `TableColumn`, `TableRow`, `TableCell` (soft-delete, fractional positions). Six column types: Text, Number, Currency, Date, Checkbox, Single Select. Core utilities: `src/core/tables/types.ts` (ColumnType, SortState, FilterState, AggregationType), `sort.ts`, `filter.ts`, `aggregations.ts` (sum/avg/count/min/max/checked_ratio), `validators.ts`, `export.ts` (JSON schema + CSV). tRPC routers: `tables` (list/get/create/update/delete, addColumn/updateColumn/reorderColumns/deleteColumn, addRow/deleteRow/reorderRows, upsertCell, search) and `tablesFolders` (list/create/rename/move/reorder/delete with cascade). UI: `src/components/tables/table-grid.tsx` (full grid: keyboard nav, inline editing, drag-to-reorder rows, column header sort/rename/move/delete, footer aggregations, add row/column), 6 cell components in `src/components/tables/cells/`, `new-table-dialog.tsx`, `table-side-panel.tsx`. Pages: `/notes/tables` (all-tables list), `/notes/tables/[tableId]` (editor with sort/filter panels), `/notes/tables/folder/[folderId]` (folder view). Notes sidebar updated with full Tables section (collapsible, folder tree, + button). Reference picker (`[[...]]`) now includes tables as a searchable target type. Search router has `search.tables` procedure. Drive-sync-tables job upgraded from stub to full export implementation (runs hourly). Audit log includes "Table" and "TablesFolder" entity types. `ProjectTablesSection` component built (ready to mount in project detail pages).
